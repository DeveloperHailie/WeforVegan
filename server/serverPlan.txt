우리가 할 기능들 (세션 관련된건 우선 대충씀)

1. 바코드 찍으면 제품명과 원재료 받아오고, 사용자의 채식타입과 매칭해서 먹으면 안되는 원재료 있는지 체크 하고 표시
 -사용자가 서버에 바코드 번호 보내면 
 -서버에서 바코드 db에서 바코드 번호에 해당하는 제품명과 원재료명, 사용자 db에서 사용자 채식타입, 먹으면 안되는 원성분 집합 인덱스가져옴 _ 근데 여기서 세션?같은거 고려해서 사용자 정보 그대로 유지하는 그런거 필요?할걸? 암튼 그거 나중에 생각하고_
 -서버에서 가져온 원재료 data로 원재료 집합 db에서 매칭되는 집합인덱스가져옴.
 -서버에서 가져온 집합인덱스와 사용자 db에서 가져온 원성분 집합 인덱스 매칭해서 먹으면 안되는성분 판단
 -서버에서 제품명, 원재료, 먹으면안되는원재료를 클라이언트에게 보냄

2. 하루 열량 고려한 식단(대표재료와 영양성분 고려하여)
 - 사용자가 하루열량 메뉴 탭하면 서버에 요청
 - 서버에서 랜덤넘버 만든 다음에
 - 서버에서 사용자db에서 사용자 나이와 성별 받아오고
 - 그정보 이용한 레시피 알고리즘 ~~
 - 그 결과를 클라이언트에게 반환

=> 여기서 말하는 레시피db가 sns레시피db말하는거
3. 레시피 검색 기능
 - 사용자가 검색어 입력 후 검색 버튼 눌러서 서버에 요청
 - 서버에서 검색어로 식품의약품안전처db, 레시피 db에서 해당 row 가져옴
 - 여기서 사진 필터링 부분 추가하면, 딥러닝 이용해서 광고인지 아닌지 체크한다음에 아닌 row들만,
 - 그리고 그 row들 중에서 출처, url, 제목 (식품의약품안전처도 이렇게 했나? 아니면 보여주는 항목 달랐나?) 을 클라이언트에게 반환
(+) 만약 사용자가 레시피 검색했을 때 좋아요 누른 레시피들 하트 채워지게 나올려면 // 밑처럼 => 세션으로
 - 서버가 사용자 db에서 좋아요한 인덱스 가져와서 검색결과와 매칭해서 
 - 하트안에 채워야하는 레시피 인덱스를 클라이언트에게 반환

4. 마이페이지 수정기능
 - 사용자가 이름, 비번, 채식 스타일, 사진 변경 후 수정하기 눌러서 서버에 요청
 - 서버에서 사용자db에서 사용자id 해당 row 변경
 - 클라이언트에게 성공 반환

5. 좋아요 기능
 - 사용자가 어떤 레시피의 좋아요 버튼 눌러서 서버에 요청
 - 서버에서 해당 레시피의 인덱스 파악해서 / 그럴려면 안드에서 인덱스를 보여주진 않아도 가지곤 있어야겠다 / 사용자 db에서 좋아요 인덱스에 추가
 - 클라이언트에게 성공 반환

(+) 좋아요 취소 기능
 (1) 레시피 검색 페이지에서 취소 했을 때 /(2) 모아보기 페이지에서 취소 했을 때
 - 사용자가 하트 취소 눌러서(레시피인덱스를 파라미터로?) 서버에 요청
 - 서버에서 사용자 id로 사용자 db에서 좋아요 누른 레시피 인덱스가져와서 혹시모를예외체크(좋아요에없는데 좋아요눌러져있다거나)
 - 서버가 사용자 db에서 사용자 id로 db에서 좋아요 누른 레시피 인덱스에서 사용자가 하트취소 누른 레시피 인덱스 제거
 - 서버가 클라이언트에게 성공 반환 

6. 좋아요 모아보기 기능
 - 사용자가 좋아요 모아보기 버튼 눌러서 서버에 요청
 - 서버에서 사용자 id 로 사용자 db에서 좋아요 누른 레시피 인덱스 가져와서
 - 레시피 db, 식품~ db에서 인덱스에 해당하는 row들 가져와서 
 - (좋아요 모아보기 화면에서 보여주는 내용에 맞게) 클라이언트에게 반환

6. 문의하기 기능
 - 사용자가 글내용 적어서 문의하기 버튼 눌러서 서버에 요청
 - 서버에서 문의하기 기능 db에 사용자id, 문의내용, 메일로 넣어
 - 클라이언트에 성공 반환 

7. 회원가입 기능
 - 사용자가 아이디, 성별, ~~ 입력 후 회원 가입 버튼 눌러서 서버에 요청
 - 서버에서 회원 db에 동일한 아이디 있는지 체크, 
 - 동일 아이디 있으면 아이디 변경하라고, 클라이언트에게 보내기
 - 동일 아이디 없으면 서버에서 사용자 db에 row추가
 - 클라이언트에 성공 반환

8. 로그인 기능
 - 사용자가 아이디, 비번 입력후 로그인 버튼 눌러서 서버에 요청
 - 서버에서 아이디, 비번 으로 사용자 db에서 row 검색후 있으면 성공, 없으면 실패 클라이언트에 반환
